在原有网页端刷题应用需求基础上，请重点增加并严格实现以下功能。

一、每次刷题都要有完整记录

每次用户开始刷题时，都要创建一次“刷题记录 session”。

无论是：
1. 练习模式
2. 测验模式
3. 错题复习模式

都要生成一条刷题记录。

每条刷题记录至少包含：

1. 记录 ID
2. 开始时间
3. 结束时间
4. 刷题模块：
   - 常见平方数
   - 常见幂次方数
   - 常见百分数
5. 刷题模式：
   - practice
   - test
   - wrongReview
6. 本次答题总数
7. 本次答对数量
8. 本次答错数量
9. 本次正确率
10. 本次答题明细列表

每一道题的答题明细至少包含：

1. 题目 ID
2. 题目文本
3. 用户输入答案
4. 正确答案
5. 是否正确
6. 答题时间
7. 所属模块
8. 所属模式

所有刷题记录保存到 localStorage 中，刷新页面后不能丢失。

请单独封装刷题记录相关逻辑，例如：

src/utils/recordStorage.ts

不要只做总统计，必须能查看每一次刷题的历史记录。

二、题目顺序必须随机

每次进入刷题时，题目顺序都要随机。

要求：

1. 不要固定按照题库顺序出题。
2. 不要简单每次 Math.random 取一题导致短时间重复过多。
3. 应该在进入刷题时，先把当前模块的题目列表打乱，形成一个随机题目队列。
4. 用户按照这个随机队列依次刷题。
5. 当前队列刷完后，如果是练习模式，可以重新洗牌继续刷。
6. 测验模式默认从随机队列中取 20 题。
7. 如果题目数量不足 20 题，可以允许重复抽题，但要尽量先完整覆盖题库。

请封装随机洗牌函数，例如：

src/utils/shuffle.ts

可以使用 Fisher-Yates 洗牌算法。

三、答题交互逻辑必须符合下面规则

刷题时的交互逻辑：

1. 用户输入答案。
2. 点击“提交答案”。
3. 如果回答正确：
   - 显示“回答正确”
   - 自动记录本题答题结果
   - 自动进入下一题
   - 可以设置短暂延迟，例如 300ms 到 600ms，不要停留太久
4. 如果回答错误：
   - 显示“回答错误”
   - 显示正确答案
   - 自动记录本题答题结果
   - 当前页面停留在本题
   - 不要自动跳到下一题
   - 必须由用户点击“下一题”后才进入下一题

注意：
正确时自动跳下一题；
错误时显示正确答案并停留，等用户自行操作。

四、必须有“刷题记录”页面

应用中必须增加一个“刷题记录”页面。

首页需要有入口：

刷题记录

刷题记录页面显示所有历史刷题 session。

每条记录显示：

1. 刷题时间
2. 刷题模块
3. 刷题模式
4. 本次答题数量
5. 正确数
6. 错误数
7. 正确率

点击某条记录后，可以查看该次刷题的详细答题明细。

明细中显示：

1. 题目
2. 我的答案
3. 正确答案
4. 正确 / 错误
5. 答题时间

五、localStorage 数据结构建议

请至少保存三类数据：

1. 总统计数据
2. 错题本数据
3. 每次刷题记录数据

建议 localStorage key：

speed_training_stats
speed_training_wrong_questions
speed_training_sessions

建议 session 数据结构：

interface AnswerRecord {
  questionId: string;
  questionText: string;
  userAnswer: string;
  correctAnswer: string;
  isCorrect: boolean;
  answeredAt: string;
  category: QuestionCategory;
  mode: PracticeMode;
}

interface PracticeSession {
  id: string;
  startedAt: string;
  endedAt?: string;
  category: QuestionCategory;
  mode: PracticeMode;
  total: number;
  correct: number;
  wrong: number;
  accuracy: number;
  answers: AnswerRecord[];
}

六、页面结构需要更新

页面至少包括：

1. 首页
2. 模块选择页
3. 练习模式页
4. 测验模式页
5. 错题复习页
6. 刷题记录页
7. 单次刷题记录详情页

七、自检要求

完成后请重点自检以下功能：

1. 每次进入练习模式都会新建刷题记录。
2. 每次进入测验模式都会新建刷题记录。
3. 每次进入错题复习都会新建刷题记录。
4. 每道题提交后都会写入本次刷题记录。
5. 正确时会自动进入下一题。
6. 错误时会显示正确答案并停留，不会自动跳题。
7. 题目顺序每次进入都不一样。
8. 刷新网页后，刷题记录仍然存在。
9. 首页可以进入刷题记录页面。
10. 刷题记录页面可以查看每一次刷题的详细记录。
11. npm run build 必须通过。
